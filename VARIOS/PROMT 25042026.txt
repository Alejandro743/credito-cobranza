LÓGICA DEL COMPONENTE LIVEWIRE:
El componente debe calcular la calificación de cartera de cada vendedor
en base a sus pedidos aprobados y el comportamiento de pago de sus clientes.

Propiedades:
- $vendedores → colección con los datos calculados
- $filtroCalificacion → string, filtra por A/B/C/D/BLOQUEADO
- $ordenar → string, ordena por puntaje desc por defecto

Método calcularVendedores():
Para cada vendedor activo con pedidos aprobados, calcular:

1. PUNTUALIDAD (peso 25%)
   - Universo: cuotas cerradas (fecha_vencimiento <= hoy)
   - A tiempo: cuotas donde fecha_pago <= fecha_vencimiento
   - Puntualidad% = (cuotas_a_tiempo / cuotas_cerradas) * 100
   - Puntos = puntualidad% directamente

2. MORA GENERADA (peso 25%)
   - Pedido en mora = tiene al menos 1 cuota cerrada sin pagar
   - Mora% = (pedidos_en_mora / total_pedidos) * 100
   - Puntos = 100 - mora%

3. CARTERA EN RIESGO (peso 20%)
   - Saldo vencido = suma de montos de cuotas cerradas sin pagar
   - Saldo pendiente = saldo vencido + suma de montos de cuotas abiertas sin pagar
   - Riesgo% = (saldo_vencido / saldo_pendiente) * 100
   - Puntos = 100 - riesgo%
   - Si saldo_pendiente = 0, riesgo = 0%

4. RECUPERACIÓN (peso 20%)
   - Solo sobre cuotas cerradas que no se pagaron originalmente
   - Monto recuperado = suma de pagos registrados sobre esas cuotas vencidas
   - Recuperacion% = (monto_recuperado / total_vencido_no_pagado) * 100
   - Puntos = recuperacion% directamente
   - Si no hay deuda vencida, recuperacion = 100%

5. REPROGRAMACIONES (peso 10%)
   - Reprogramado% = (pedidos_reprogramados / total_pedidos) * 100
   - Puntos = 100 - reprogramado%

FÓRMULA FINAL:
puntaje = (punt * 0.25) + ((100-mora) * 0.25) + ((100-riesgo) * 0.20) + (recup * 0.20) + ((100-reprog) * 0.10)

TABLA DE CALIFICACIÓN:
- 85-100 → A (Excelente)
- 70-84  → B (Buena)
- 50-69  → C (Observada)
- 30-49  → D (Riesgosa)
- 0-29   → BLOQUEADO
